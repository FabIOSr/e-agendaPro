Checklist final
Do código ao ar em ~45 minutos

Este guia cobre tudo desde criar a conta no Supabase até seu app funcionando com domínio próprio no Firebase Hosting. Siga as fases em ordem — cada uma depende da anterior.

ℹ️ Tempo estimado: ~45 min na primeira vez. Deploys futuros: menos de 5 min.
1
Supabase — Banco de dados
Criar projeto, rodar migrations e configurar RLS
1
Criar conta e projeto no Supabase
Leva menos de 2 minutos. É gratuito para começar.
▼
  1. Acesse supabase.com → clique em Start your project
  2. Faça login com GitHub (recomendado) ou e-mail
  3. Clique em New project
  4. Escolha um nome (ex: agendapro), uma senha forte para o banco e a região South America (São Paulo)
  5. Clique em Create new project e aguarde ~2 minutos
💡Guarde a senha do banco — você vai precisar dela para conexões diretas (ex: Prisma, DBeaver).
2
Copiar credenciais do projeto
Você vai precisar da URL e das chaves para configurar o front-end.
▼

No painel do Supabase, vá em Settings → API e copie:

  • Project URL — ex: https://xyzabc.supabase.co
  • anon public — chave pública usada no front-end
  • service_role — chave secreta usada nas Edge Functions (nunca expor no front)
⚠️A service_role tem acesso total ao banco, ignorando RLS. Só use em Edge Functions com variável de ambiente.
3
Rodar as migrations (8 arquivos, em ordem)
Cria todas as tabelas, funções, triggers e políticas RLS.
▼

No painel Supabase vá em SQL Editor → New query. Cole e execute cada arquivo abaixo na ordem indicada (um de cada vez):

01 01_auth.sql Trigger de criação de perfil + RLS
02 02_assinaturas.sql Tabela pagamentos + colunas Asaas
03 03_cron_downgrade.sql Cron de downgrade + função verifica_plano_ativo
04 04_cancel_token.sql cancel_token nos agendamentos
05 05_avaliacoes_gcal.sql Tabelas avaliacoes + google_calendar_tokens
06 06_crm_clientes.sql Tabela clientes + view crm_clientes
07 07_receita_functions.sql Funções SQL do relatório de receita
08 08_avaliacao_batch.sql Flag avaliacao_solicitada + cron batch
💡Se der erro em alguma migration, leia a mensagem. O erro mais comum é rodar fora de ordem. Sempre execute na sequência 01 → 08.
2
Supabase — Autenticação
Configurar login com e-mail e Google OAuth
1
Configurar URL do site
Informa ao Supabase para onde redirecionar após login.
▼

Vá em Authentication → URL Configuration:

Site URL:
https://seuapp.com

Redirect URLs (adicionar todas):
https://seuapp.com/pages/painel.html
https://seuapp.com/pages/auth.html
http://localhost:5000/pages/painel.html   ← para testes locais
2
Habilitar Google OAuth
Permite que prestadores se cadastrem com a conta Google.
▼
  1. Acesse console.cloud.google.com → crie um projeto novo
  2. Vá em APIs & Services → Credentials → Create credentials → OAuth Client ID
  3. Tipo: Web application
  4. Authorized redirect URI — copie do painel Supabase em Auth → Providers → Google → Callback URL:
    https://xyzabc.supabase.co/auth/v1/callback
  5. Copie o Client ID e Client Secret
  6. De volta no Supabase: Auth → Providers → Google → Enable → cole as credenciais
💡O Google OAuth também é necessário para o Google Calendar. Configure uma vez e serve para os dois.
3
Personalizar e-mail de confirmação
Opcional mas recomendado para maior profissionalismo.
▼

Vá em Authentication → Email Templates → Confirm signup e substitua o template pelo conteúdo abaixo:

<h2>Confirme seu e-mail — AgendaPro</h2>
<p>Olá! Clique no link abaixo para ativar sua conta:</p>
<p><a href="{{ .ConfirmationURL }}">Ativar minha conta</a></p>
<p style="color:#888;font-size:12px">Se não foi você, ignore este e-mail.</p>
3
Edge Functions
Instalar Supabase CLI e fazer deploy das 10 funções
1
Instalar o Supabase CLI
Ferramenta de linha de comando para deploy das Edge Functions.
▼ 
# macOS (Homebrew)
brew install supabase/tap/supabase

# Windows (Scoop)
scoop bucket add supabase https://github.com/supabase/scoop-bucket.git
scoop install supabase

# Linux / WSL
curl -fsSL https://deno.land/install.sh | sh  # Deno (runtime das funções)
npx supabase --version  # ou via npm
2
Login e link com o projeto
Conecta o CLI ao seu projeto Supabase.
▼ 
# Login (abre o browser)
supabase login

# Link com seu projeto
# (o Project ID está em Settings → General)
supabase link --project-ref SEU_PROJECT_ID
3
Deploy de todas as Edge Functions
Um comando só faz o deploy das 10 funções.
▼

Na raiz do projeto (pasta agendapro/):

for fn in horarios-disponiveis lembretes-whatsapp \
          criar-assinatura webhook-asaas cancelar-assinatura \
          cancelar-agendamento-cliente reagendar-cliente \
          avaliacoes solicitar-avaliacao-batch google-calendar-sync
do
  supabase functions deploy $fn
done
💡Para atualizar uma função específica depois: supabase functions deploy nome-da-funcao
4
Variáveis de ambiente (Secrets)
Configurar todas as chaves externas nas Edge Functions
1
Configurar todos os secrets de uma vez
Cole e edite o bloco abaixo com suas chaves reais.
▼ 
supabase secrets set \
  ZAPI_INSTANCE_ID="SUA_INSTANCIA_ZAPI" \
  ZAPI_TOKEN="SEU_TOKEN_ZAPI" \
  ZAPI_CLIENT_TOKEN="SEU_CLIENT_TOKEN_ZAPI" \
  ASAAS_API_KEY="$aas_XXXXXXXXXXXX" \
  ASAAS_WEBHOOK_TOKEN="token-secreto-que-voce-escolhe" \
  GOOGLE_CLIENT_ID="XXXXXXXXX.apps.googleusercontent.com" \
  GOOGLE_CLIENT_SECRET="GOCSPX-XXXXXXXXXX" \
  APP_URL="https://seuapp.com"
Onde obter cada secret:
ZAPI_* z-api.io → Sua instância → Aba "Token"
ASAAS_API_KEY asaas.com → Minha Conta → Integrações → API
ASAAS_WEBHOOK_TOKEN Você escolhe — qualquer string longa e aleatória
GOOGLE_* console.cloud.google.com → Credentials → seu OAuth Client
2
Configurar webhook do Asaas
Conecta o Asaas à Edge Function webhook-asaas.
▼
  1. Acesse asaas.com → Minha Conta → Integrações → Webhooks
  2. Clique em Novo webhook
  3. URL: https://SEU_PROJETO.supabase.co/functions/v1/webhook-asaas
  4. Token: o mesmo valor de ASAAS_WEBHOOK_TOKEN
  5. Eventos: marque todos — especialmente PAYMENT_RECEIVED, PAYMENT_OVERDUE, SUBSCRIPTION_DELETED, PAYMENT_REFUNDED
5
Firebase Hosting
Publicar as páginas HTML com HTTPS automático
1
Criar projeto no Firebase
Interface visual — sem linha de comando por enquanto.
▼
  1. Acesse console.firebase.google.com
  2. Clique em Add project → nome: agendapro
  3. Desmarque Google Analytics (desnecessário no começo)
  4. Aguarde criação (~30 segundos)
2
Instalar Firebase CLI e fazer login
Node.js precisa estar instalado (node.js.org).
▼ 
# Instala globalmente
npm install -g firebase-tools

# Login (abre o browser)
firebase login

# Verifica
firebase --version
3
Substituir as variáveis no código
Antes do deploy, substitua os placeholders nos arquivos.
▼

Faça um busca e substituição em todos os arquivos da pasta pages/ e modules/: 

# macOS / Linux — substitui em todos os arquivos de uma vez:
find agendapro/pages agendapro/modules -type f \
  -exec sed -i '' \
    -e 's|https://SEU_PROJETO.supabase.co|https://XYZABC.supabase.co|g' \
    -e 's|SUA_ANON_KEY|eyJhbGci...|g' \
  {} \;

# Windows (PowerShell):
Get-ChildItem -Recurse -Include *.html,*.js | ForEach-Object {
  (Get-Content $_) -replace 'SEU_PROJETO', 'XYZABC' | Set-Content $_
}
⚠️Substitua XYZABC pelo seu Project ID real e eyJhbGci... pela sua anon key real.
4
Inicializar e fazer deploy
Configura o Firebase Hosting e sobe os arquivos.
▼ 
# Na raiz da pasta agendapro/
firebase init hosting

# Responda as perguntas:
# ? What do you want to use as your public directory? → agendapro
# ? Configure as a single-page app? → No
# ? Set up automatic builds with GitHub? → No (por enquanto)
# ? File agendapro/index.html already exists. Overwrite? → No
# Deploy!
firebase deploy --only hosting

# Saída esperada:
# ✔  Deploy complete!
# Hosting URL: https://agendapro-xxxxx.web.app
🎉Seu app já está no ar em https://agendapro-xxxxx.web.app! Para deploys futuros, só rode firebase deploy.
5
Configurar rewrites de rotas
Para que /painel abra painel.html, /auth abra auth.html etc.
▼

Crie (ou edite) o arquivo firebase.json na raiz:

{
  "hosting": {
    "public": "agendapro",
    "ignore": ["firebase.json", "**/.*", "**/node_modules/**"],
    "rewrites": [
      { "source": "/",            "destination": "/pages/landing-page.html" },
      { "source": "/auth",        "destination": "/pages/auth.html" },
      { "source": "/onboarding",  "destination": "/pages/onboarding.html" },
      { "source": "/painel",      "destination": "/pages/painel.html" },
      { "source": "/planos",      "destination": "/pages/planos.html" },
      { "source": "/clientes",    "destination": "/pages/clientes.html" },
      { "source": "/relatorio",   "destination": "/pages/relatorio.html" },
      { "source": "/configuracoes","destination": "/pages/configuracoes.html" },
      { "source": "/cancelar",    "destination": "/pages/cancelar-cliente.html" },
      { "source": "/reagendar",   "destination": "/pages/reagendar-cliente.html" },
      { "source": "/avaliar",     "destination": "/pages/avaliar-cliente.html" },
      { "source": "/:slug",       "destination": "/pages/pagina-cliente.html" }
    ]
  }
}
# Redeploy com os rewrites
firebase deploy --only hosting
6
Domínio personalizado
Conectar seu domínio (ex: agendapro.com.br) ao Firebase
1
Adicionar domínio no Firebase
Firebase gera SSL automático com Let's Encrypt.
▼
  1. No Firebase Console → Hosting → Add custom domain
  2. Digite seu domínio: agendapro.com.br
  3. O Firebase mostrará dois registros TXT para verificar a propriedade
  4. Adicione esses registros no painel do seu registrador de domínio (Registro.br, GoDaddy, Namecheap…)
  5. Aguarde até 48h para propagação (geralmente menos de 1h)
  6. Após verificado, o Firebase mostrará dois registros A — adicione-os também no DNS
  7. SSL é ativado automaticamente em até 24h ✅
💡Adicione também www.agendapro.com.br com redirecionamento para o domínio principal.
2
Atualizar URLs no Supabase
Após conectar o domínio, atualize as URLs de redirecionamento.
▼

Volte em Authentication → URL Configuration e atualize:

Site URL:
https://agendapro.com.br

Redirect URLs:
https://agendapro.com.br/painel
https://agendapro.com.br/auth

E atualize também o Google Cloud Console → OAuth → Authorized redirect URIs para incluir o novo domínio.
